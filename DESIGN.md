# Design System: ทะเบียนควบคุมเอกสารคุณภาพ

## 1. Overview

**Creative North Star: "The Clinical Logbook" (สมุดบันทึกทางคลินิกอันเที่ยงตรง)**

ระบบทะเบียนเอกสารคุณภาพได้รับการออกแบบให้มีภาพลักษณ์ที่ "แม่นยำ · น่าเชื่อถือ · สงบ" เพื่อตอบรับการทำงานอย่างเป็นระบบภายใต้มาตรฐานสากล ISO 15189:2022 หลีกเลี่ยงรูปแบบการออกแบบตามกระแสสตาร์ทอัพ SaaS (เช่น สีสันฉูดฉาด, เงาการ์ดลอยหนา, กราเดียนต์สะดุดตา) และเน้นความเรียบง่าย เป็นสัดส่วน ชัดเจน น่าเชื่อถือ คล้ายสมุดบันทึกสถิติระดับสถาบันที่เป็นเอกสารราชการยุคใหม่ 

การสลับธีมมีความเท่าเทียมกันในการรองรับทั้งการใช้งานบนเดสก์ท็อปในสำนักงานกลางวัน (ครีม) และสมาร์ทโฟนของเจ้าหน้าที่ระหว่างปฏิบัติหน้าที่กะกลางคืนในห้องปฏิบัติการ (มืด) การแบ่งลำดับความสำคัญของเนื้อหาอย่างชัดเจนช่วยลดภาระทางความคิด (Cognitive Load) ให้กับผู้ปฏิบัติงานในสภาวะงานที่เร่งรีบ

**Key Characteristics:**
- **Monospace Numbers**: รหัสเอกสาร รหัสเวอร์ชัน และข้อมูลรหัสควบคุมเด่นด้วยฟอนต์ประเภท Monospace เพื่อการค้นหาและตรวจสอบได้อย่างรวดเร็ว
- **Flat Layout Boundaries**: การจำกัดส่วนเนื้อหาด้วยเส้นขอบบางและการจับคู่สีพื้นหลัง (Contrast Ratios) แทนการใช้เงาลอย เพื่อรักษาความเสถียรและบรรยากาศที่เป็นทางการ
- **Two Equal Themes**: ธีมมืดและธีมสว่าง (ครีม) ใช้สถาปัตยกรรม CSS Variables เดียวกันทั้งหมด ทำให้การนำไปปฏิบัติใช้งานบนหน้าจอมือถือจริงมีความเสมอภาคในทุกๆ จุด

## 2. Colors

ชุดสีหลักเน้นสีเขียวสัญลักษณ์ของสถาบัน (Thammasat Green) ร่วมกับโทนสว่างและโทนมืดที่มีความเปรียบต่างสูง (Contrast Ratio ≥ 4.5:1) เพื่อการอ่านที่ชัดเจน

### Primary
- **Thammasat Green** (`#37b26e` ในธีมมืด / `#1e7a46` ในธีมครีม): ใช้เป็นสีหลักในการเน้นสถานะประกาศใช้ (ACTIVE) และการดำเนินการหลักของระบบ เช่น ปุ่มบันทึกการอ่าน/รับทราบ
- **Thammasat Green Deep** (`#2a8f58` ในธีมมืด / `#166337` ในธีมครีม): ใช้สำหรับการโต้ตอบในสถานะ Hover หรือ Active ของปุ่มทำงานหลัก

### Neutral
- **Background** (`#0a0d0c` ในธีมมืด / `#f4f1e8` ในธีมครีม): พื้นผิวพื้นหลังของหน้าระบบหลัก
- **Background Low** (`#0e1211` ในธีมมืด / `#ece8db` ในธีมครีม): พื้นผิวสำหรับแถบข้อมูลหรือโครงร่างภายนอก
- **Surface** (`#111615` ในธีมมืด / `#faf8f1` ในธีมครีม): สีพื้นหลังของการ์ด แบบฟอร์ม และป็อปอัปโมดัล
- **Surface 2** (`#161c1a` ในธีมมืด / `#efebdf` ในธีมครีม): พื้นหลังปุ่มทำงานหรือรายการที่ต้องการการจัดกลุ่มพิเศษ
- **Ink Primary** (`#eff3f1` ในธีมมืด / `#171f1a` ในธีมครีม): สีตัวอักษรเนื้อความหลัก
- **Ink Secondary** (`#a9b4af` ในธีมมืด / `#3d4a43` ในธีมครีม): สีตัวอักษรข้อมูลรอง
- **Ink Muted** (`#7e8a84` ในธีมมืด / `#5a675f` ในธีมครีม): สีตัวอักษรคำบรรยายย่อยและข้อความช่วยเหลือ

### Status Colors
- **Active (Green)** (`#37b26e` ในธีมมืด / `#1e7a46` ในธีมครีม): แสดงสถานะเอกสาร "ประกาศใช้"
- **Review (Blue)** (`#6ba0d6` ในธีมมืด / `#33628f` ในธีมครีม): แสดงสถานะเอกสาร "ระหว่างทบทวน"
- **Draft (Amber)** (`#d8a441` ในธีมมืด / `#9e7013` ในธีมครีม): แสดงสถานะเอกสาร "ฉบับร่าง" หรือรายการรอรับทราบการอ่าน
- **Obsolete (Red)** (`#d2695f` ในธีมมืด / `#b04a40` ในธีมครีม): แสดงสถานะเอกสาร "ยกเลิกใช้"

### Named Rules
**The 10% Accent Rule.** สีเขียวสัญลักษณ์จะถูกใช้เฉพาะบนสิ่งที่มีความสำคัญต่อขั้นตอนการทำเอกสารเท่านั้น (เช่น ปุ่มบันทึกรับทราบการอ่านหลัก จุดสถานะประกาศใช้ หรือรหัสเอกสารเด่น) พื้นที่กว่า 90% ของระบบต้องรักษาโทน Neutral เพื่อความสงบและน่าเชื่อถือ

## 3. Typography

ระบบทะเบียนนี้ใช้ฟอนต์ภาษาไทยไม่มีหัวสไตล์กึ่งทางการเป็นส่วนหัวข้อ และฟอนต์มีหัวสไตล์มาตรฐานสำหรับข้อมูลข้อความทั่วไปเพื่อให้สอดคล้องกับรูปแบบเอกสารควบคุมทางราชการ

**Display Font:** Anuphan (ฟอนต์ไม่มีหัวที่ให้ความรู้สึกโมเดิร์น กระชับ และเป็นระเบียบ)
**Body Font:** Sarabun (ฟอนต์มีหัวที่ผ่านการยอมรับในการใช้งานเอกสารราชการไทย อ่านง่าย มีระเบียบ)
**Label/Mono Font:** IBM Plex Mono (สำหรับรหัส เวอร์ชัน ตัวเลข และสัญลักษณ์ทางเทคนิค)

**Character:** การจับคู่ระหว่าง Anuphan และ Sarabun ช่วยดึงลักษณะ "ราชการยุคใหม่" ออกมาอย่างชัดเจน ให้บรรยากาศน่าเชื่อถือ น่าอ่าน และถูกต้องเที่ยงตรง

### Hierarchy
- **Display** (Weight 600, Size `clamp(1.5rem, 3vw, 2.3rem)`, Line-height 1.14): สำหรับหัวข้อหน้าหลัก (Page Titles) และชื่อเอกสารคุณภาพ
- **Headline** (Weight 600, Size 18px, Line-height 1.3): สำหรับหัวข้อส่วนข้อมูลหรือหัวข้อสำคัญในการดำเนินงาน
- **Body** (Weight 400, Size 15.5px, Line-height 1.55): สำหรับข้อความอธิบายข้อมูล ประวัติการแก้ไข และคู่มือการใช้งานหลัก
- **Label / Monospace** (Weight 400, Size 13px, Letter-spacing 0.03em): สำหรับรหัสเอกสาร (เช่น `HEM-WI-001`), เวอร์ชัน (v.01), และ Audit Logs ต่างๆ

### Named Rules
**The Readable Prose Rule.** ข้อความประเภทเนื้อความทั่วไป (Body) ทั้งปวงจะต้องถูกจำกัดความยาวต่อบรรทัดให้อยู่ในขอบเขต 65–75 ตัวอักษร (65-75ch) เพื่อให้อ่านข้อมูลเอกสารได้อย่างแม่นยำที่สุดโดยไม่ล้าสายตา

## 4. Elevation

ระบบทะเบียนนี้ออกแบบในแนวทาง **Flat Design (แบนราบไร้มิติเงา)** โดยสมบูรณ์แบบ หลีกเลี่ยงเงาหนาของการ์ดลอย เพื่อป้องกันลักษณะโมเดลแอปพลิเคชันสตาร์ทอัพเชิงพาณิชย์

การสร้างมิติและความลึก (Depth) จะถูกระบุด้วย:
- **Border Partitioning**: เส้นแบ่งขนาดบาง 1px (`var(--line)` และ `var(--line2)`) เพื่อกั้นพื้นที่องค์ประกอบ
- **Background Tonal Contrast**: การสลับใช้พื้นผิวระบายสีต่างกัน เช่น เอาการ์ดสี Surface (`#111615`) ไปลอยทับบนตัวบอดี้พื้นหลังสี Background (`#0a0d0c`)
- **Interaction Fills**: การเปลี่ยนสีเมื่อโต้ตอบกับปุ่ม เช่น เปลี่ยนพื้นหลังเป็น Surface 2 เมื่อมีการ Hover ปลายเมาส์

### Named Rules
**The Absolute Flatness Rule.** ห้ามใส่เงาลอย (`box-shadow`) บนการ์ด ตาราง หรือปุ่มต่างๆ นอกเหนือไปจากโมดัลป็อปอัปที่ระบุสิทธิ์ฉุกเฉิน ซึ่งใช้เงาประเภทกระจายตัวละเอียดเพื่อทับซ้อนชั้นเลเยอร์การทำงานปกติเท่านั้น

## 5. Components

### Buttons
- **Shape:** ขอบตัดเหลี่ยมเกือบสนิท รัศมีความโค้งมนเล็กมาก (2px radius) เพื่อความรู้สึกมั่นคง แม่นยำ
- **Primary Button:** พื้นหลังสีเขียว Accent ตัวหนังสือสีดำสนิท (`#05130c` หรือ `var(--accent-ink)`) ห้ามมีเส้นขอบใดๆ
- **Ghost/Outline Button:** พื้นหลังโปร่งใส เส้นขอบบาง (`var(--line2)`) สีตัวหนังสือตามธีมหลัก เหมาะกับการเข้าถึงเมนูรอง
- **Danger Button:** เส้นขอบสีแดงจางๆ ตัวหนังสือสีแดง เหมาะสำหรับการดำเนินการที่ไม่สามารถแก้ไขย้อนคืนได้ เช่น ลบดราฟท์หรือยกเลิกการใช้งานเอกสาร

### Chips / Badges
- **Style:** รูปร่างตัดเหลี่ยมเกือบมน มีความโค้งมนรัศมีเท่าปุ่ม (2px radius)
- **Status Badges:** เป็นป้ายแสดงรหัสสถานะเอกสาร ใช้เส้นขอบตามสีของสถานะนั้นๆ (ACTIVE=เขียว, REVIEW=ฟ้า, DRAFT=เหลือง, OBSOLETE=แดง) ควบคู่กับจุดสีนำสายตา 6px ทางซ้ายมือ ห้ามใช้พื้นหลังทึบในระดับสถานะปกติเพื่อรักษาความสะอาดตา

### Cards / Containers
- **Border:** เส้นขอบ 1px สี `var(--line2)` ล้อมรอบพื้นที่ทั้งหมด
- **Background:** ใช้สี Surface เสมอเพื่อเพิ่มความเปรียบต่างกับพื้นหลังบอดี้

### Inputs / Fields
- **Style:** พื้นหลังสีเข้มกว่าปกติ (`var(--bg2)`) ล้อมรอบด้วยเส้นขอบ `var(--line2)` ตัดขอบเหลี่ยมแบบละเอียด
- **Focus:** ขยายความชัดของเส้นขอบด้วยสี Accent เขียวสว่าง โดยไม่มีการแสดงเงารืองแสงรอบนอกกล่องข้อความ

## 6. Do's and Don'ts

### Do:
- **Do** แสดงรหัสเอกสารนำหน้าชื่อเอกสารเสมอในทุกๆ ลิงก์และลิสต์ เพื่อเอื้อต่อการค้นหา
- **Do** ใช้ป้ายกำกับข้อความร่วมกับไอคอนต่างๆ เสมอ (ห้ามใช้ไอคอนเดี่ยวลอยๆ โดยไม่มีข้อความอธิบายการใช้งาน)
- **Do** ออกแบบให้ Flow การเข้าถึงการกดยอมรับเอกสารคุณภาพ สามารถกดปุ่มอ่านและบันทึกการรับทราบได้เสร็จสมบูรณ์บนจอมือถือขนาดหน้ากว้าง 375px ในระดับความสูง 2-Scroll
- **Do** คำนึงถึงการลดการเคลื่อนไหว โดยปิดการเลื่อนไหวหรือการเฟดข้อมูลทั้งหมดเมื่อตรวจพบสิทธิ์ `@media (prefers-reduced-motion: reduce)`

### Don't:
- **Don't** ใช้สไตล์การ์ดซ้อนการ์ด (Nested Cards) เด็ดขาด ให้จัดการจัดวางสถาปัตยกรรมข้อมูลเป็นคอลัมน์แบนราบหรือแถวข้อมูลแทน
- **Don't** ใช้การแรเงาขอบข้าง (Side-stripe Borders) ในสีอื่นๆ เพื่อไฮไลต์การ์ดหรือกล่องแจ้งเตือนภัย
- **Don't** เขียนส่วนข้อความ Eyebrow ขนาดเล็กที่เป็นตัวพิมพ์ใหญ่ (All Caps) ทั้งหมดซ้ำกันเป็นเทมเพลตในทุกช่วงเนื้อหา (เช่นคำว่า "ABOUT" "PROCESS" "PRICING")
- **Don't** ตกแต่งตัวหนังสือด้วยกราเดียนต์สี (Gradient Text) 
- **Don't** ผสมสไตล์กล่องเงาเบลอแบบกระจก (Glassmorphic surfaces)
- **Don't** ใช้ภาษาที่ไม่เป็นทางการหรือใส่ไอคอนอีโมจิแนวบันเทิงในการอธิบายตัวเอกสารควบคุม เพื่อให้ระบบมีความสุภาพ สงบ และถูกต้องสมบูรณ์สูงสุด
