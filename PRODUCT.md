# Product

## Register

product

## Users

เจ้าหน้าที่ห้องปฏิบัติการฝ่ายสหเวชศาสตร์ โรงพยาบาลธรรมศาสตร์เฉลิมพระเกียรติ (~150 คน, 7 บทบาท: ผู้ดูแลระบบ, หัวหน้างาน, หัวหน้าหมวดงาน, นักเทคนิคการแพทย์, ผู้ช่วยนักเทคนิคการแพทย์, เจ้าหน้าที่ธุรการ, ผู้จัดการเอกสาร) ใช้งานระหว่างเวรปฏิบัติงานในห้องแล็บ ทั้งบนเดสก์ท็อปหน้างานและมือถือส่วนตัว ภาษาไทยเป็นหลัก งานหลักคือค้นหา/อ่าน/รับทราบเอกสารคุณภาพ ส่วนผู้มีสิทธิ์สูงกว่าใช้ลงทะเบียน แก้ไข ประกาศใช้ และแนบไฟล์

## Product Purpose

ทะเบียนควบคุมเอกสารคุณภาพห้องปฏิบัติการ (Quality Document Masterlist) ตามมาตรฐาน ISO 15189:2022 — แทนที่ทะเบียนกระดาษ/สเปรดชีตด้วยระบบเดียวที่ค้นหาเร็ว มีวงจรเอกสารครบ (ร่าง→ทบทวน→ประกาศใช้→ยกเลิก) บันทึกการรับทราบรายบุคคล และเก็บ audit trail สำหรับการตรวจประเมิน สำเร็จเมื่อเจ้าหน้าที่ทุกคนหาเอกสารฉบับปัจจุบันเจอภายในไม่กี่วินาที และผู้ตรวจประเมินย้อนรอยได้ทุกการกระทำ

## Brand Personality

แม่นยำ · น่าเชื่อถือ · สงบ — บุคลิกแบบ "เอกสารราชการยุคใหม่": เขียว institutional ของโรงพยาบาล, ตัวเลข/รหัสใช้ monospace, จังหวะเรียบนิ่งไม่หวือหวา ให้ความรู้สึกระบบคุณภาพที่ไว้ใจได้ ไม่ใช่ startup SaaS

## Anti-references

- SaaS dashboard สีสันจัดจ้าน gradient หนา ๆ การ์ดลอยเงาหนา
- ระบบราชการยุคเก่า: ตาราง HTML ดิบ ฟอนต์ Angsana พื้นหลังขาวจ้าไร้ลำดับชั้น
- Consumer app โทนสนุกสนาน อีโมจิเยอะ ภาษาไม่เป็นทางการ

## Design Principles

1. **รหัสเอกสารคือพระเอก** — code (HEM-WI-001) เด่นด้วย monospace + accent เสมอ ผู้ใช้สแกนหารหัสก่อนชื่อ
2. **สถานะต้องอ่านได้ในแวบเดียว** — จุดสี + ป้ายสถานะสม่ำเสมอทั้งระบบ (เขียว=ประกาศใช้, ฟ้า=ทบทวน, เหลือง=ร่าง, แดง=ยกเลิก)
3. **ทุกการกระทำมีร่องรอย** — UI สื่อว่าระบบบันทึกทุกอย่าง (ผู้ทำ วันที่ เวอร์ชัน) เพื่อความเชื่อมั่นตาม ISO
4. **สองธีมเท่าเทียม** — dark (ห้องแล็บกลางคืน) และ cream (สำนักงานกลางวัน) ต้องอ่านชัดทั้งคู่ ทุก element ใช้ CSS variables
5. **มือถือคือหน้างานจริง** — เจ้าหน้าที่เปิดมือถือรับทราบเอกสารระหว่างเวร ทุก flow ต้องจบได้บนจอ 375px

## Accessibility & Inclusion

ภาษาไทยเป็นหลัก (ฟอนต์ Sarabun/Anuphan รองรับ) · contrast ≥4.5:1 ทั้งสองธีม · prefers-reduced-motion ปิดอนิเมชันทั้งหมด · ผู้ใช้หลากหลายวัยและความคุ้นเคยกับเทคโนโลยี — UI ต้องอธิบายตัวเอง ปุ่มมีข้อความกำกับเสมอไม่ใช้ไอคอนลอย ๆ
